# Gomoku

## Processus d'approbation des Pull Requests ✅

Pour clarifier notre flux de travail lors des revues de code, une règle simple s'applique :

- **Poster une image ou un GIF dans la conversation d'une Pull Request est considéré comme une approbation explicite** de la PR.

Quelques précisions :
- Utilisez cette méthode uniquement si vous êtes d'accord avec les changements proposés. Si vous avez des réserves, laissez plutôt un commentaire écrit expliquant vos doutes.
- Si vous devez retirer votre approbation, précisez-le clairement avec un commentaire (par exemple "Retrait d'approbation"); le fait de poster une image ne retire pas une approbation.
- Pour des changements critiques, privilégiez la revue formelle (bouton "Approve" sur GitHub) ou un commentaire écrit détaillé.

Merci de respecter cette convention afin d'améliorer la communication et la rapidité de nos revues.

## Pull Request Approval Process ✅

To clarify our code review workflow, one simple rule applies:

- **Posting an image or GIF in the discussion of a Pull Request is considered an explicit approval of the PR.**

A few clarifications:
- Use this method only if you agree with the proposed changes. If you have concerns, leave a written comment explaining them instead.
- If you need to revoke your approval, clearly state that in a comment (for example "Approval revoked"); posting an image does not remove an approval.
- For critical changes, favor a formal review (the "Approve" button on GitHub) or a detailed written review.

Thank you for following this convention to improve communication and speed up our reviews.
